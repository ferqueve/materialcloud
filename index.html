Conceptos Básicos de Cloud Computing

SaaS, PaaS, e IaaS: Los Pilares de la Nube

Modelos de Servicio en la Nube

  • SaaS: Software as a Service
  • PaaS: Platform as a Service
  • IaaS: Infrastructure as a Service

Cada modelo ofrece diferentes niveles de control y responsabilidad para el usuario.

SaaS (Software as a Service)

Software entregado a través de internet, sin necesidad de instalación local.

Características:
  • Accesible vía web browser
  • No requiere mantenimiento por parte del usuario
  • Actualizaciones automáticas
  • Ejemplos: Google Workspace, Salesforce, Dropbox

PaaS (Platform as a Service)

Plataforma para desarrollar, ejecutar y gestionar aplicaciones sin la complejidad de mantener la infraestructura.

Características:
  • Herramientas de desarrollo y despliegue
  • Gestión de bases de datos
  • Escalabilidad automática
  • Ejemplos: Google App Engine, Heroku, Microsoft Azure App Service

IaaS (Infrastructure as a Service)

Recursos de computación, almacenamiento y redes bajo demanda.

Características:
  • Máquinas virtuales
  • Almacenamiento
  • Redes
  • Máximo control sobre los recursos
  • Ejemplos: Amazon EC2, Google Compute Engine, Microsoft Azure VMs

Ventajas del Cloud Computing

  • Reducción de costos de infraestructura
  • Escalabilidad y flexibilidad
  • Accesibilidad global
  • Actualizaciones automáticas
  • Recuperación ante desastres
  • Seguridad mejorada

Bucket

Un contenedor para almacenar objetos en servicios de almacenamiento en la nube.

  • Nombre único a nivel global
  • Almacenamiento ilimitado de objetos
  • Control de versiones y políticas de acceso

Cluster

Grupo de servidores o instancias que trabajan juntos como un sistema único.

Tipos comunes:
  • Clusters de alta disponibilidad
  • Clusters de balanceo de carga
  • Clusters de computación para big data

Job

Tarea o proceso que se ejecuta en los recursos de la nube.

  • Tienen un inicio y un fin definidos
  • Pueden ser programados o activados por eventos
  • Comunes en procesamiento de big data

Instance

Máquina virtual en la nube, esencialmente un servidor virtual.

  • Escalables vertical y horizontalmente
  • Diferentes configuraciones de recursos
  • Pueden ser efímeras o persistentes

Container

Unidad estándar de software que empaqueta código y dependencias.

  • Ligereza y eficiencia
  • Portabilidad entre entornos
  • Aislamiento de aplicaciones

Load Balancer

Distribuye el tráfico de red o solicitudes entre servidores o instancias.

  • Mejora disponibilidad y capacidad de respuesta
  • Realiza comprobaciones de salud
  • Maneja cifrado SSL/TLS

VPC (Virtual Private Cloud)

Red virtual aislada lógicamente en la nube pública.

  • Mayor seguridad y aislamiento
  • Control sobre la topología de red
  • Conexión segura con redes on-premises

Responsabilidades en la Nube

División de tareas de gestión y seguridad entre el proveedor de la nube y el cliente.

  • Varía según el modelo de servicio (IaaS, PaaS, SaaS)
  • El cliente siempre es responsable de sus datos
  • La seguridad es una responsabilidad compartida

Pub/Sub (Publicación/Suscripción)

Patrón de mensajería para comunicación asíncrona entre servicios.

  • Publishers envían mensajes a temas
  • Subscribers reciben mensajes de temas a los que están suscritos
  • Desacopla componentes de la aplicación

Cloud Functions

Pequeñas piezas de código que se ejecutan en respuesta a eventos.

  • Sin necesidad de gestionar infraestructura
  • Escalado automático
  • Pago solo por el tiempo de ejecución

Deployments (Despliegues)

Proceso de poner una aplicación o servicio en producción en la nube.

  • Incluye configuración y distribución del código
  • Puede ser manual o automatizado (CI/CD)
  • Estrategias: rolling update, blue-green, canary

Containers (Contenedores)

Unidades estándar de software que empaquetan código y dependencias.

  • Portabilidad entre diferentes entornos
  • Aislamiento de aplicaciones
  • Eficiencia en recursos comparado con VMs

Serverless Computing

Modelo donde el proveedor gestiona dinámicamente la infraestructura.

  • El desarrollador se enfoca solo en el código
  • Escalado automático y alta disponibilidad
  • Pago por uso, no por capacidad reservada
